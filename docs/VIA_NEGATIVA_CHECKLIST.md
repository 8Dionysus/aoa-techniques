# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-techniques` as the reusable practice canon.

## Keep intact

- techniques that are truly reusable across repos
- clear trigger boundaries, risks, and verification guidance
- one canonical technique per materially distinct pattern family

## Merge, move, suppress, quarantine, deprecate, or remove when found

- near-duplicate techniques with different names and nearly identical steps
- techniques that are mostly repo-local runbook fragments
- large philosophy preambles repeated across many technique docs

## Questions before adding anything new

1. Is the distinction between these techniques real enough to survive six
   months of use?
2. Would one canonical technique plus repo-local examples be cleaner?
3. Is this actually a skill or owner-local procedure rather than a reusable
   technique?

## Safe exceptions

- a pattern whose substrate or safety posture genuinely differs
- tiny helper notes kept local rather than published as full techniques

## Exit condition

- Technique count should rise slower than meaning density.
