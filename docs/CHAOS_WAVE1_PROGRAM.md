## Core move

This wave is not a random-failure playground.

It is a bounded stress program that teaches the system to:

- fail legibly
- degrade honestly
- route conservatively
- keep blocked widening explicit
- hand evidence upward without hiding a judge inside runtime

## Core flow

1. inject or encounter one bounded stress family
2. emit one owner-local receipt or equivalent artifact
3. keep the degraded continuation honest or hold the route explicitly
4. block unsafe widening
5. route the next hop to the owning layer that should read it next
6. let derived summaries appear only after owner evidence exists
7. let bounded proof reading happen later in `aoa-evals`
8. close out with reviewed, bounded repair artifacts

## Families in scope

### timeout chaos

The system proves it can hit a bounded timeout without pretending success,
widening repair, or losing operator visibility.

### honest degradation chaos

The system proves it can continue in a weaker mode and say so explicitly.

### retrieval outage honesty

When a derived or retrieval-bearing surface is unhealthy, consumers fall back
to source-first or held posture instead of fabricating continuity.

### skill collision chaos

Activation surfaces prove that strong semantic overlap still resolves to one of
the honest outcomes:

- manual invocation when explicit-only
- defer-to-other-skill when the neighbor owns the task
- no-skill when the shortlist is weak or the intent is still unclear

### trace-integrity chaos

The route proves that path evidence, trace surfaces, and verification artifacts
stay coherent enough for later bounded proof reading.

### evidence bridge

Runtime or route artifacts may become review candidates for `aoa-evals`, but
runtime does not become the judge.
