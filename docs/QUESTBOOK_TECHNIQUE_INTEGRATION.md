# QUESTBOOK integration — aoa-techniques

## Purpose

This note shows how `QUESTBOOK.md` fits into `aoa-techniques` without turning the repo into a second donor backlog.

## Role split

- `docs/START_HERE.md` stays the repo-owned entrypoint
- `TECHNIQUE_INDEX.md` remains the public technique surface
- technique files remain the source of technique meaning
- donor notes and candidate intake remain intake or review material until a deferred obligation survives the current diff
- `QUESTBOOK.md` tracks deferred obligations such as canon hardening, donor-refinery follow-through, generated/source drift, and harvest candidates

## Good anchors in this repo

Use stable anchors such as:
- `docs/PROMOTION_READINESS_MATRIX.md`
- `docs/CROSS_LAYER_TECHNIQUE_CANDIDATES.md`
- `docs/DONOR_REFINERY_RUBRIC.md`
- `generated/technique_capsules.min.json`
- `docs/KAG_EXPORT.md`
- `generated/repo_doc_surface_manifest.json`

## Initial posture

A technique quest should normally point back to one of these:
- a promotion gate
- a donor refinement debt
- a generated/source drift seam
- a repeated review route that wants canon promotion

Do not mint a quest for every donor note.
Mint one only when the obligation survives the current bounded change and needs future routing, proof, or harvest.
