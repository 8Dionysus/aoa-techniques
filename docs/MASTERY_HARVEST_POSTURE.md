# Mastery Harvest Posture

## Purpose

This note explains how repeated routes and evidence surfaces in `aoa-techniques` can nominate or stabilize feat-shaped mastery reflections.

It does not define a promotion shortcut. It defines a bounded reading posture.

## Honest nomination sources

Good second-wave nomination sources include:

- donor-refinery work that survives into reusable canon
- repeated review and evidence-note repair routes
- second-context adaptation that keeps the core contract intact
- external evidence or portability checks
- generated/source alignment work that exposes a stable reusable edge

These signals can nominate a feat card. They do not, by themselves, rewrite technique status.

## Honest outputs now

In this wave, harvest may produce:

- a feat card example
- a quest nomination for later canon-hardening
- stronger mastery wording around transfer scope and caution posture

It should not produce:

- automatic canonical promotion
- automatic skill creation
- a combo engine
- a leaderboard or permanent rank system inside `aoa-techniques`

## Transfer scope

Name transfer scope carefully:

- `repo_local` when the reusable edge is only proven inside one repo family
- `cross_repo` when the edge is already shown to travel between repos
- `cross_layer` when the edge proves legible across neighboring AoA layers

Overclaiming transfer scope is worse than leaving it narrow.

## Final rule

Let harvest deepen reusable-practice legibility. Do not let it smuggle in score worship or bypass source-owned review.
