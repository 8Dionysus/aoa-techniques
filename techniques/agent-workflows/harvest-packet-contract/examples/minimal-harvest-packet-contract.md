# minimal harvest-packet-contract

```yaml
harvest_packet:
  session_ref: reviewed-session-2026-04-05
  reviewed_artifacts:
    - compaction-note-2026-04-05.md
  extracts:
    - title: explicit post-session donor harvest workflow
      kind: mechanic
      summary: keep donor extraction visible before later placement or promotion
      evidence_refs:
        - "the reviewed note repeatedly turns raw recap into bounded reusable extracts"
      repeat_signal: repeated in the reviewed session artifact
      owner_hint: aoa-skills
      chosen_next_surface: SKILL.md
      nearest_wrong_target: aoa-routing
  fork_cards:
    - "optional later branch analysis"
  progression:
    - "optional later progression overlay"
```

Why this example stays bounded:

- it keeps the required packet spine small
- it records one bounded extract instead of one long recap
- it treats later family fields as optional pointers rather than replacements for sibling seams
