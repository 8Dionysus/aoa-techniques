# harvest-packet-contract checklist

- [ ] the packet starts from reviewed material rather than live session state
- [ ] the required packet spine is explicit and small
- [ ] each extract record names one bounded reusable unit
- [ ] optional family fields stay subordinate rather than becoming hidden verdicts
- [ ] the packet does not replace donor extraction, owner placement, diagnosis, repair, progression, or quest seams
- [ ] the example stays readable as a bounded packet rather than a recap blob
- [ ] the public wording stays reusable and sanitized
