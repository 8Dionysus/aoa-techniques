# Second Context Adaptation

## Technique
- id: AOA-T-0078
- name: decision-fork-cards

## Target project
- name: aoa-techniques
- environment: public documentation repository for reusable engineering techniques
- runtime: human and agent contribution workflow over portable technique bundles

## What changed
- the skill-local route-forks behavior became one bounded branch-card technique
- AoA repo naming and quest-board framing moved into adaptation notes instead of the invariant core
- route passports stayed separate as the sibling `risk-passport-lift` technique instead of remaining fused into one broad bundle
- the bundle was reduced to one technique doc, one checklist, one example, and three evidence notes

## What stayed invariant
- the branch cards still start only after review
- each card still has to be materially distinct
- gains, costs, risks, and stop conditions still stay visible
- the output still stays advisory instead of authoritative

## Risks introduced by adaptation
- public wording can drift into generic brainstorming if material branch difference is not kept explicit
- the bundle can widen into routing policy if examples start hiding a preferred path as authority
- a minimal example can understate the need for hold or defer posture

## Evidence
- the adapted bundle stays in `agent-workflows` because the reusable object is one bounded post-session workflow for explicit branch analysis
- the public wording still preserves the seam between branch cards and smaller route-passport metadata
- the origin evidence remains strong enough to justify a promoted public bundle without importing local skill wrappers or repo-local routing tables

## Result
- verdict: works
- note: the adapted bundle stays readable as a bounded branch-card workflow over reviewed next routes
