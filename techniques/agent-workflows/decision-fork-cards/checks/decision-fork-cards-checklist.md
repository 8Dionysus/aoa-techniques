# decision-fork-cards checklist

- [ ] the source material is already reviewed
- [ ] each card represents a materially different route
- [ ] each route names at least one likely gain, cost, and risk
- [ ] likely first owner targets are visible
- [ ] risky routes include explicit stop conditions
- [ ] hold, defer, or reanchor remains available when uncertainty is real
- [ ] the output stays advisory rather than becoming hidden routing policy
- [ ] the public wording stays reusable and sanitized
