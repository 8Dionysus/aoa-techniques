# minimal decision-fork-cards

```yaml
fork_cards:
  - route: strengthen the donor nucleus first
    likely_gain: clearer packet contract before any wider rollout
    likely_cost: one extra focused hardening pass
    likely_risk: delayed delivery of downstream siblings
    first_owner_target: aoa-skills
    stop_condition: "stop if the nucleus changes no longer fit one bounded skill family"
  - route: extract the diagnosis seam next
    likely_gain: repeated drift becomes explicit before repair
    likely_cost: one more technique and skill pinning pass
    likely_risk: diagnosis may outrun real usage proof
    first_owner_target: aoa-techniques
    stop_condition: "stop if diagnosis starts smuggling repair work"
  - route: hold
    likely_gain: avoids forced branching when evidence is still mixed
    likely_cost: slower visible momentum
    likely_risk: uncertainty may linger
    first_owner_target: hold
    stop_condition: "reopen once reviewed evidence sharpens"
```

Why this example stays bounded:

- each card represents a materially distinct next route
- costs, risks, and stop conditions stay visible
- the card set preserves choice instead of hiding one default route as policy
