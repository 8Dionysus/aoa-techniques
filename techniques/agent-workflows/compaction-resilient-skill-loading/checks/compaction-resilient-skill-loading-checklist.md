# compaction-resilient-skill-loading checklist

- [ ] the compaction boundary is explicit
- [ ] canonical skill sources remain named and authoritative
- [ ] post-compaction recovery restores a bounded availability surface rather than full prompt history
- [ ] needed skills can still be reloaded from canonical sources after compaction
- [ ] restored capability state is visible enough to review
- [ ] the bundle stays smaller than deterministic context composition
- [ ] the bundle stays out of marketplace, install, and memory doctrine
- [ ] the example keeps the recovery path public-safe and concrete
