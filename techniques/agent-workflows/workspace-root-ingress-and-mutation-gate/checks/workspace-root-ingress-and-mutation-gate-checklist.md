# workspace-root-ingress-and-mutation-gate checklist

- [ ] the workspace root is named explicitly
- [ ] the primary repo root is named explicitly
- [ ] ingress runs before substantial work begins
- [ ] guard runs before risky mutation begins
- [ ] `must_confirm` and `blocked_actions` remain visible and actionable
- [ ] the technique stays smaller than install, bootstrap, or closeout doctrine
- [ ] the public wording stays reusable and sanitized
