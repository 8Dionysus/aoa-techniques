# minimal session-donor-harvest

## Reviewed session artifact cues

- one repeated post-session move for turning review notes into a bounded donor packet
- one explicit operator workflow for harvesting reusable units after review
- one broader recurring rollout route that spans drafting, review, and later proof updates

## Minimal donor pack

```yaml
session_donor_pack:
  source_artifact: reviewed-session-2026-04-05.md
  kept_candidates:
    - unit_name: reviewed-session donor packet
      reuse_kind: pattern
      evidence_anchors:
        - "the review note repeatedly turns raw recap into a bounded candidate pack"
    - unit_name: explicit post-session donor harvest workflow
      reuse_kind: mechanic
      evidence_anchors:
        - "the operator runs one bounded post-session harvesting workflow after review"
    - unit_name: recurring harvest-to-promotion route
      reuse_kind: route
      evidence_anchors:
        - "the route spans drafting, review, and later proof refresh"
  hold_or_defer:
    - "theme-level morale notes with no bounded reusable unit"
```

Why this example stays bounded:

- it starts from a reviewed artifact instead of a live session
- it emits candidate records with evidence anchors rather than a promotion verdict
- it preserves several reusable units without collapsing them into one recap or one owner-layer answer
