# git-verified-handoff-claims checklist

- [ ] the handoff contains at least one concrete claim with file, commit, or diff anchors
- [ ] visible git evidence is checked before continuation trusts the handoff
- [ ] the verification output names the exact evidence used
- [ ] mismatches and unverifiable claims stay explicit
- [ ] the example stays smaller than generic code review, witness artifacts, or provenance doctrine
- [ ] no specific orchestrator, branch policy, or snapshot system is required to understand the contract
- [ ] the bundle stays smaller than the live `phase-synchronized-agent-handoff` narrowing lane
- [ ] the public wording remains reusable and sanitized
