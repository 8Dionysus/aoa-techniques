# minimal owner-layer-triage

## Candidate unit

```yaml
candidate_unit:
  unit_name: explicit post-session donor harvest workflow
  reuse_kind: mechanic
  evidence_anchors:
    - "the reviewed session repeatedly performs one bounded post-session operator workflow"
```

## Placement verdict

```yaml
owner_layer_verdict:
  primary_owner_shape: bounded_executable_workflow
  next_artifact: SKILL.md
  rejected_nearest_wrong_target:
    owner_shape: reusable_practice_canon
    reason: "the value here depends on a bounded operator workflow rather than on one abstract rule alone"
```

Why this example stays bounded:

- it starts from one already isolated candidate unit
- it chooses one primary owner verdict rather than several partial owners
- it rejects the nearest wrong target explicitly instead of leaving the boundary implicit
