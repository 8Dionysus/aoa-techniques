# Second Context Adaptation

## Technique
- id: AOA-T-0076
- name: owner-layer-triage

## Target project
- name: aoa-techniques
- environment: public documentation repository for reusable engineering techniques
- runtime: human and agent contribution workflow over portable technique bundles

## What changed
- AoA repo names were reduced to owner-shape language in the invariant core
- the donor skill's local owner map became one portable placement technique with adaptation notes instead of a runtime-only reference sheet
- donor extraction was split into the sibling `session-donor-harvest` technique so this bundle only owns one owner verdict per candidate
- the bundle was reduced to one technique doc, one checklist, one example, and three evidence notes

## What stayed invariant
- one bounded candidate unit still maps to one primary owner layer
- the nearest wrong target still has to be rejected explicitly
- the chosen next artifact still remains proportional to the owner verdict
- derivative routing or graph surfaces still stay out of first-authoring

## Risks introduced by adaptation
- the public wording can drift back into ecosystem-specific routing if the adaptation notes overshadow the core owner-shape contract
- the bundle can widen into context-map doctrine if examples start explaining whole repo topologies instead of one bounded placement verdict
- a minimal example can understate the need to split mixed units before triage

## Evidence
- the adapted bundle stays in `agent-workflows` because the reusable object is one bounded placement workflow, not a docs taxonomy or routing system
- the public wording still keeps one candidate, one owner, one rejected nearest-wrong target as the core seam
- the donor evidence remains strong enough to justify a promoted public bundle without importing the whole AoA routing surface

## Result
- verdict: works
- note: the adapted bundle stays readable as a bounded workflow for owner-layer placement over one reusable unit
