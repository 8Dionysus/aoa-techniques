# owner-layer-triage checklist

- [ ] the candidate unit is already split into one bounded reusable unit
- [ ] one primary owner layer is named explicitly
- [ ] the nearest wrong target is rejected explicitly
- [ ] the chosen next artifact is proportional to the owner verdict
- [ ] derivative routing or graph surfaces were not treated as first-authoring targets
- [ ] `hold` remains available when the evidence is weak or mixed
- [ ] the example stays smaller than full context-map, promotion-board, or routing doctrine
- [ ] the public wording stays reusable and sanitized
