# session-opening-ritual-before-work checklist

- [ ] the ritual happens before the first mutation in a resumed or handed-off session
- [ ] at least one current context surface is read explicitly
- [ ] at least one visible baseline check is performed against current state
- [ ] any mismatch between expected and actual state is named explicitly
- [ ] the first honest move can be explained from the verified baseline
- [ ] the example stays smaller than handoff authoring, git-claim verification, task picking, or startup governance doctrine
- [ ] the bundle does not depend on one orchestrator runtime, state-file family, or launchd-style supervision
- [ ] the public wording remains reusable and sanitized
