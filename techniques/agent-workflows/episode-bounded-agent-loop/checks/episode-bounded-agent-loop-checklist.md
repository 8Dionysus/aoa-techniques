# episode-bounded-agent-loop checklist

- [ ] each episode has a bounded goal
- [ ] each episode reaches a visible checkpoint or stop condition
- [ ] continue, stop, or escalate remains an explicit decision
- [ ] the next episode can explain its start from the checkpointed state
- [ ] the example stays smaller than orchestration platforms, supervision stacks, or total autonomous-agent doctrine
- [ ] the bundle does not depend on one runtime, budget cap, task-tracker contract, or launchd-style supervisor
- [ ] checkpoint artifacts remain reviewable enough for another reader to follow
- [ ] the public wording remains reusable and sanitized
