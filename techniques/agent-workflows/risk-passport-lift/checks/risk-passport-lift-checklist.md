# risk-passport-lift checklist

- [ ] each passport belongs to one explicit route
- [ ] the field set stays small and reviewable
- [ ] difficulty, risk, control mode, and stop-condition cues remain visible where needed
- [ ] stop conditions exist for meaningfully risky routes
- [ ] the passport does not become hidden routing authority
- [ ] the example stays smaller than the branch card it complements
- [ ] the public wording stays reusable and sanitized
