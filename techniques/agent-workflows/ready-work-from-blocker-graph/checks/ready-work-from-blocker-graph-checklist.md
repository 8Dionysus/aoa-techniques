# ready-work-from-blocker-graph checklist

Use this checklist when reviewing whether a ready-work frontier still reflects blocker truth instead of slipping into hidden overrides or broad prioritization policy.

- [ ] queue derivation depends on blocker-free graph state
- [ ] blocked tasks keep visible reasons for exclusion
- [ ] open-but-blocked work does not appear in the ready queue
- [ ] any secondary ordering happens only after readiness is established
- [ ] manual overrides, if any, stay explicit rather than hidden
- [ ] the queue remains explainable from visible inputs
- [ ] graph authoring and broader PM policy stay outside this technique
- [ ] at least one example shows the frontier changing after a blocker clears
