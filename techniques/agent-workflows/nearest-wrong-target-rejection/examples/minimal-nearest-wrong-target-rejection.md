# minimal nearest-wrong-target-rejection

```yaml
target_rejection:
  chosen_target: promote_to_playbook
  nearest_wrong_target: promote_to_skill
  rejection_reason: repeated unit is route-shaped, not one leaf workflow
```

Why this example stays bounded:

- it rejects the closest wrong target instead of a distant strawman
- it uses one boundary reason rather than a long essay
- it clarifies the chosen verdict without replacing the verdict itself
