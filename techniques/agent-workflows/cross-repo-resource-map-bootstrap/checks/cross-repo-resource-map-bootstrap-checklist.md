# cross-repo-resource-map-bootstrap checklist

- [ ] the map is tied to one current cross-repo task or handoff
- [ ] every listed repo has an explicit role
- [ ] every listed resource or surface has a stated reason to be in the map
- [ ] the map shows where the next reader should look first
- [ ] the map stays small enough to inspect quickly
- [ ] the example stays smaller than bounded-context mapping, architecture inventory, or workspace-platform doctrine
- [ ] the bundle does not depend on one global orchestrator path, project board, or collaboration mode
- [ ] the public wording remains reusable and sanitized
