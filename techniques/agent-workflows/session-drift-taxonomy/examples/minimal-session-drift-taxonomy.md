# minimal session-drift-taxonomy

```yaml
session_drift_types:
  - type: boundary_drift
    evidence: "technique language and skill language kept collapsing together"
  - type: proof_debt
    evidence: "review claims were repeated without stable proof anchors"
  - type: route_collapse
    evidence: "several possible follow-ups kept being compressed into one vague next step"
```

Why this example stays bounded:

- it classifies drift types before claiming one probable cause
- it keeps the output smaller than a full diagnosis packet
- it leaves owner hints and repair shapes for later seams
