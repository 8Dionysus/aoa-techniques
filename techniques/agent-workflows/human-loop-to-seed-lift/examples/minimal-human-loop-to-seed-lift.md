# minimal human-loop-to-seed-lift

```yaml
seed_lift:
  route: reviewed session closeout loop
  owner_layer: aoa-playbooks
  next_artifact: playbook_seed
  nearest_wrong_target: live_scheduler
  schedule_hint: nightly-after-review
```

Why this example stays bounded:

- it names one first honest landing instead of implementing the automation
- it keeps the schedule hint advisory rather than sovereign
- it preserves one rejected wrong target so promotion pressure stays explicit
