# Witness Trace As Reviewable Artifact Checklist

- [ ] The artifact names one bounded run, goal, and review scope.
- [ ] The trace remains an export artifact rather than a memory object or taxonomy addition.
- [ ] Ordered steps remain visible, and tool visibility is preserved when present.
- [ ] State deltas are recorded when an external effect occurred.
- [ ] One compact human-readable summary explains what happened and what should be inspected next.
- [ ] Redaction-first handling removes secret-bearing payloads and hidden chain-of-thought detail.
- [ ] Failure-path or review-flag preservation remains visible when the run did not go cleanly.
- [ ] Memory writeback, compost promotion, eval anchors, and role choreography stay outside the core contract.
- [ ] The bundle stays distinct from [AOA-T-0026](../session-capture-as-repo-artifact/TECHNIQUE.md) and [AOA-T-0044](../versionable-session-transcripts/TECHNIQUE.md).
