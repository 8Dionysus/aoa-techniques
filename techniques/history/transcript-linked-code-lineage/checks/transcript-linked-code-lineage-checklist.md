# transcript-linked-code-lineage checklist

- [ ] the lineage surface starts from already-saved evidence artifacts
- [ ] code anchors such as files, line ranges, or commits point back to stable evidence references
- [ ] another reviewer can reopen the saved evidence from the lineage surface
- [ ] saved evidence remains authoritative rather than being replaced by the lineage pointer
- [ ] the example stays smaller than repo analytics, scorecards, or hosted search platforms
- [ ] handoff verification, productivity telemetry, and free-form Q and A are not treated as the invariant core
- [ ] the wording does not collapse lineage into memory or generic retrieval doctrine
- [ ] the public wording remains reusable and sanitized
