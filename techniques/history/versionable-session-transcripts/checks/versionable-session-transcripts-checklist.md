# Versionable Session Transcripts Checklist

- [ ] The transcript package starts from already-saved session artifacts rather than from first-save capture.
- [ ] One bounded review or handoff unit selects one conversation or one intentional group of conversations.
- [ ] The exported transcript stays readable as Markdown or an equivalently versionable text artifact.
- [ ] Timestamps, metadata, or naming make the transcript package stable enough for commit, comparison, or citation.
- [ ] Review, edit, annotation, or redaction can happen before wider sharing without changing the artifact into a summary or rule file.
- [ ] The technique stays distinct from [AOA-T-0026](../session-capture-as-repo-artifact/TECHNIQUE.md) and does not re-own capture or persistence.
- [ ] Hosted share links, cloud search, login flows, and sync status are not treated as the core contract.
- [ ] History-to-instructions, derived-rules, and memory recall semantics remain outside the bundle.
- [ ] The exported transcript does not become the canonical source of truth for repository instructions or policy.
