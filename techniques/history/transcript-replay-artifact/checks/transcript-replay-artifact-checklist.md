# transcript-replay-artifact checklist

- [ ] replay starts from an already-saved session or transcript artifact
- [ ] the replay package preserves bounded flow cues such as message order or timeline position
- [ ] the artifact keeps a stable reference back to the saved source session
- [ ] replay stays derivative rather than becoming the authoritative history layer
- [ ] the example stays smaller than transcript packaging, witness tracing, or hosted replay products
- [ ] publish flows, dashboards, and live monitoring are not treated as the invariant core
- [ ] the wording does not reopen capture or memory doctrine
- [ ] the public wording remains reusable and sanitized
