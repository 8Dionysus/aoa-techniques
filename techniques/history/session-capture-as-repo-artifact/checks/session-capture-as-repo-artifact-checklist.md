# Session Capture As Repo Artifact Checklist

- Save sessions into one project-scoped artifact location.
- Keep local persistence available even when cloud sync is absent.
- Preserve enough structure that sessions can be inspected or summarized later.
- Keep captured history separate from authored instructions and decision docs.
- Do not widen the contract into memory recall, vector retrieval, or policy routing.
- Sanitize or review session artifacts before treating them as portable or shareable.
- Keep search, analytics, and history-derived skills as separate layers outside this technique.
