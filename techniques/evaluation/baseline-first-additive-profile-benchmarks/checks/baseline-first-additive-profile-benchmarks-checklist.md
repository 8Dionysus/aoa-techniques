# Baseline First Additive Profile Benchmarks Checklist

- [ ] The baseline profile runs first.
- [ ] Additive profiles are compared only after the baseline result exists.
- [ ] The same measurement surface is used for the baseline and additive runs.
- [ ] The same artifact shape is produced for the baseline and additive runs.
- [ ] Additive prework stays isolated from the default path.
- [ ] The result reports comparison discipline rather than product scoring.
- [ ] The benchmark does not claim ownership of suite governance or promotion policy.
- [ ] The benchmark does not drift into rolling historical baseline logic.
- [ ] The benchmark stays separate from profile composition, render review, host diagnosis, and lifecycle control.
- [ ] The public wording remains portable and does not depend on donor-specific service names.
