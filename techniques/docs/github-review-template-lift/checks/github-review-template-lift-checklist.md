# GitHub Review Template Lift Checklist

Use this checklist to confirm that GitHub review templates still behave like a derived intake surface rather than a decision engine.

- The lifted template scopes come from authored `.github` markdown files.
- The derived surface matches the current template inventory.
- The manifest answers "which review prompt should I open?" rather than "what should the outcome be?"
- Template meaning still lives in the authored markdown files.
- No approval policy, triage engine, or review-state registry was needed.
- The surface stays bounded to issue and pull-request review intake.
