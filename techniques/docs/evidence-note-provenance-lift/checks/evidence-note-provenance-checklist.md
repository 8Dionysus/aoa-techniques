# Evidence Note Provenance Checklist

Use this checklist to confirm that note-level provenance stays bounded and markdown-first.

- Each supporting note is reachable through explicit `evidence.kind` and `evidence.path`.
- Note roles are small, understandable, and review-shaped.
- The derived provenance surface is rebuilt from authored notes instead of edited directly.
- Opaque notes stay opaque unless a reusable typed shape is truly stable.
- The workflow does not require note IDs, a merged note graph, or proof-object semantics.
