# Nested Rule Loading Checklist

- Name one canonical source that owns shared rule meaning.
- Define the parent and nested layers before relying on the loaded result.
- State the precedence order explicitly.
- Verify that nested layers stay subordinate to the canonical source.
- Confirm that the resolved output is repeatable from the same hierarchy.
- Check that nested layers add scoped meaning rather than becoming new sources of truth.
- Keep multi-target distribution, MCP propagation, and skills propagation out of the contract.
