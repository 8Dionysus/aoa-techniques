# capability-discovery checklist

- [ ] discovery operates over already-published entries or records
- [ ] query fields are explicit and bounded
- [ ] match behavior is readable and does not depend on hidden runtime logic
- [ ] result shape is explicit about whether it returns identifiers, records, or both
- [ ] the example stays smaller than ranking, marketplace curation, trust policy, or registry-product doctrine
- [ ] query fields do not smuggle in graph semantics or governance policy
- [ ] changes to discovery fields or response semantics can be reviewed at the contract layer
- [ ] the public wording remains reusable and sanitized
