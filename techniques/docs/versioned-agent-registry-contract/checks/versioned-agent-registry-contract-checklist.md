# versioned-agent-registry-contract checklist

- [ ] the entry identity includes an explicit name, version, and stable record reference
- [ ] schema version or equivalent bounded metadata is visible
- [ ] the entry contract is distinguishable from the underlying record payload
- [ ] the example stays smaller than discovery queries, marketplace curation, or registry-product doctrine
- [ ] annotations or metadata fields stay bounded and reviewable
- [ ] the bundle does not depend on one search API, trust pipeline, or distributed runtime
- [ ] version changes can be reviewed at the entry-contract layer
- [ ] the public wording remains reusable and sanitized
