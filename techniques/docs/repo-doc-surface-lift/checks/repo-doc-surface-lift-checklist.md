# Repo Doc Surface Lift Checklist

Use this checklist to confirm the repo-doc surface still behaves like bounded routing knowledge.

- The source set is explicitly bounded to the public docs/status files.
- `TODO.md`, `PLANS.md`, and `ROADMAP.md` stay out of the source class.
- Deeper guide docs and review docs stay out unless they belong to a separate source class.
- The reader surface points to authored docs rather than replacing them.
- The surface answers "which doc should I open?" instead of trying to explain the full content of the doc.
- The derived surface can be rebuilt from markdown source.
- No status-policy engine or catch-all docs taxonomy was introduced.
