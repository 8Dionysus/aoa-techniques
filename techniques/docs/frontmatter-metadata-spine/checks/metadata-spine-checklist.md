# Metadata Spine Checklist

Use this checklist to confirm that frontmatter still behaves like a bounded metadata spine rather than a shadow copy of the full technique.

- Frontmatter exposes only identity, routing, review, evidence, and direct-adjacency handles.
- The derived catalog is rebuilt from source frontmatter instead of edited by hand.
- The catalog answers bounded routing questions such as status, domain, or direct relation lookup.
- Questions about section meaning, caution language, or provenance still route back to markdown files.
- No new `kag` domain, richer schema, or metadata-first source-of-truth claim was needed.
